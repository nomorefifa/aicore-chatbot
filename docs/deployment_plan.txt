# 강사 매칭 챗봇 — 배포 및 확장 계획

---

## 전체 구성도

```
개발자 로컬
    ↓ git push
GitHub (코드 저장소)
    ↓ push 감지 → 자동 실행
GitHub Actions (CI/CD)
    ↓ Docker 빌드 + 푸시
Artifact Registry (GCP 이미지 저장소)
    ↓ 새 이미지로 자동 교체
Cloud Run (서버 — HTTPS 자동 발급)
    ↓
직원들 브라우저에서 접속
```

데이터(ChromaDB)는 GCS(Cloud Storage)에 별도 저장.
코드와 데이터를 분리하여 관리.

---

## 1단계 — GitHub 저장소 설정

### 목적
- 코드 버전 관리
- CI/CD 파이프라인 트리거 포인트

### 작업 내용

1. GitHub에 private 저장소 생성

2. .gitignore 설정 — 아래 항목 제외
   - .env (API 키)
   - data/vector_db/ (ChromaDB 파일 — GCS에서 관리)
   - data/raw/ (원본 파일 — 민감 정보)
   - data/parsed/ (파싱 결과 JSON)
   - __pycache__/, *.pyc

3. 브랜치 전략
   - main → 운영 배포 (push 시 자동 배포)
   - dev  → 개발 작업 (PR 머지 시 main 반영)

### 초기 push 명령어
```
git init
git remote add origin https://github.com/your-org/instructor-agent.git
git add .
git commit -m "init"
git push origin main
```

## 2단계 — GCP 프로젝트 셋업

### 필요한 GCP 서비스

| 서비스 | 용도 |
|--------|------|
| Cloud Run | 앱 서버 (컨테이너 실행) |
| Artifact Registry | Docker 이미지 저장소 |
| Cloud Storage (GCS) | ChromaDB vector_db 저장 |
| Secret Manager | API 키 등 민감 정보 관리 |

### 활성화할 API
```
gcloud services enable run.googleapis.com
gcloud services enable artifactregistry.googleapis.com
gcloud services enable storage.googleapis.com
gcloud services enable secretmanager.googleapis.com
```

### GCS 버킷 생성 (vector_db 저장용)
```
gsutil mb -l asia-northeast3 gs://your-project-vector-db/
```

### Secret Manager에 API 키 등록
```
echo -n "your_google_api_key" | \
  gcloud secrets create GOOGLE_API_KEY --data-file=-
```

## 3단계 — Dockerfile 작성

### 구성 방식
- Python 이미지 기반
- 코드 복사 + 패키지 설치
- Cloud Run 시작 시 GCS에서 vector_db 자동 다운로드
- Gradio 앱 실행

### Dockerfile
```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# GCS 동기화용 gsutil 설치
RUN apt-get update && apt-get install -y curl gnupg && \
    echo "deb [signed-by=/usr/share/keyrings/cloud.google.gpg] \
    https://packages.cloud.google.com/apt cloud-sdk main" | \
    tee /etc/apt/sources.list.d/google-cloud-sdk.list && \
    curl https://packages.cloud.google.com/apt/doc/apt-key.gpg | \
    apt-key --keyring /usr/share/keyrings/cloud.google.gpg add - && \
    apt-get update && apt-get install -y google-cloud-cli

COPY . .

# 시작 시 GCS에서 vector_db 가져온 후 앱 실행
CMD gsutil -m cp -r gs://your-project-vector-db/vector_db/ data/ && \
    python app.py
```

### app.py 수정 (포트 설정)
```python
# Cloud Run은 PORT 환경변수로 포트를 지정함
import os
demo.launch(server_name="0.0.0.0", server_port=int(os.environ.get("PORT", 7860)))
```

## 4단계 — 수동 배포 1회 (검증용)

CI/CD 연결 전 수동으로 먼저 배포해서 정상 동작 확인.

```bash
# 이미지 빌드
docker build -t asia-northeast3-docker.pkg.dev/YOUR_PROJECT/instructor/app:latest .

# Artifact Registry에 푸시
docker push asia-northeast3-docker.pkg.dev/YOUR_PROJECT/instructor/app:latest

# Cloud Run 배포
gcloud run deploy instructor-agent \
  --image asia-northeast3-docker.pkg.dev/YOUR_PROJECT/instructor/app:latest \
  --region asia-northeast3 \
  --platform managed \
  --set-secrets GOOGLE_API_KEY=GOOGLE_API_KEY:latest \
  --memory 2Gi \
  --timeout 300
```

## 5단계 — GitHub Actions CI/CD 파이프라인

### 파일 위치
.github/workflows/deploy.yml

### 파이프라인 내용
```yaml
name: Deploy to Cloud Run

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v4

      - name: GCP 인증
        uses: google-github-actions/auth@v2
        with:
          credentials_json: ${{ secrets.GCP_SA_KEY }}

      - name: Docker 이미지 빌드 및 푸시
        run: |
          gcloud auth configure-docker asia-northeast3-docker.pkg.dev
          docker build -t asia-northeast3-docker.pkg.dev/${{ secrets.GCP_PROJECT }}/instructor/app:${{ github.sha }} .
          docker push asia-northeast3-docker.pkg.dev/${{ secrets.GCP_PROJECT }}/instructor/app:${{ github.sha }}

      - name: Cloud Run 배포
        run: |
          gcloud run deploy instructor-agent \
            --image asia-northeast3-docker.pkg.dev/${{ secrets.GCP_PROJECT }}/instructor/app:${{ github.sha }} \
            --region asia-northeast3 \
            --platform managed \
            --set-secrets GOOGLE_API_KEY=GOOGLE_API_KEY:latest \
            --memory 2Gi
```

### GitHub Secrets 등록 (저장소 Settings → Secrets)
| 키 | 값 |
|----|-----|
| GCP_SA_KEY | GCP 서비스 계정 JSON 키 |
| GCP_PROJECT | GCP 프로젝트 ID |

### 개발자 워크플로우
```
코드 수정
    → git push origin main
    → GitHub Actions 자동 실행 (약 3~5분)
    → 직원들이 같은 URL로 새 버전 접속
```

## 6단계 — 직원 접근 권한 설정

### 옵션 A — URL 공유 (빠르고 단순)
Cloud Run을 Public으로 열고 URL만 내부 공유.
외부에서도 URL 알면 접속 가능하므로 보안 주의.

### 옵션 B — Google Workspace 계정 인증 (권장)
회사 Google 계정(@yourcompany.com)으로만 접속 가능.

```bash
# 도메인 전체에 접근 허용
gcloud run services add-iam-policy-binding instructor-agent \
  --region asia-northeast3 \
  --member="domain:yourcompany.com" \
  --role="roles/run.invoker"
```

접속 시 Google 로그인 화면 → 회사 계정으로 인증 → 챗봇 화면.

## 7단계 — 새 강사 파일 추가 시 워크플로우

코드 배포와 데이터 업데이트는 분리되어 있음.

```
1. 새 파일을 data/raw/ 또는 data/raw_new/ 에 추가

2. 로컬에서 임베딩 실행
   python test/pdf_parsing.py

3. 결과물을 GCS에 업로드
   gsutil -m cp -r data/vector_db/ gs://your-project-vector-db/

4. Cloud Run 인스턴스 재시작 (새 데이터 반영)
   gcloud run services update instructor-agent \
     --region asia-northeast3 \
     --update-env-vars REFRESH=true
```

재배포(이미지 재빌드) 없이 데이터만 갱신 가능.

## 8단계 — 파일 보안 등급별 분기 처리

### 배경
- 강사 이력서, 공개 교육자료 → 외부 API(Gemini) 사용 가능
- 계약서, 내부 평가자료, 급여 정보 → 외부 전송 금지, 로컬 모델만 사용

### 오픈소스 대체 스택

| 역할 | 외부 API (공개 파일) | 오픈소스 (기밀 파일) |
|------|---------------------|---------------------|
| 구조화 파싱 | Gemini Flash | Ollama + llama3 / qwen2.5 |
| 임베딩 | gemini-embedding-001 | sentence-transformers (BGE-M3 등) |
| 실행 환경 | 외부 API 호출 | 로컬 또는 GCP VM 내부 |

### 폴더 구조로 보안 등급 분리
```
data/
├── raw/
│   ├── public/    → Gemini 사용 가능 (이력서, 공개자료)
│   └── private/   → 로컬 모델만 사용 (계약서, 내부자료)
```

### 코드 구조 (추가될 파일)
```
src/parsing/
├── parser_factory.py      # 보안 등급 보고 파서 자동 선택
├── local_parser.py        # Ollama 기반 로컬 파서
src/embedding/
├── embedder_factory.py    # 보안 등급 보고 임베더 자동 선택
├── local_embedder.py      # sentence-transformers 기반 로컬 임베더
```

### 분기 처리 흐름
```
파일 경로 입력
    ↓
parser_factory.py가 경로 확인
    ├── public/ 폴더 → ResumeParser (Gemini API)
    └── private/ 폴더 → LocalResumeParser (Ollama, 외부 전송 없음)
    ↓
embedder_factory.py가 동일하게 분기
    ├── public/ → EmbeddingStore (Gemini Embedding)
    └── private/ → LocalEmbeddingStore (sentence-transformers)
    ↓
동일한 ChromaDB에 저장 (컬렉션만 분리)
```

### 로컬 모델 실행 환경 (GCP 기준)
```
GCP Compute Engine VM (CPU or GPU)
    └── Ollama 서버 실행
          └── qwen2.5 / llama3 로드
                ↓
          Cloud Run → GCP 내부 네트워크로 호출
          (인터넷 밖으로 데이터 절대 안 나감)
```

sentence-transformers 임베딩은 GPU 없이 CPU만으로도 동작 가능.
한국어 품질은 BGE-M3 또는 qwen2.5 계열 권장.

## 전체 진행 순서 요약

| 순서 | 작업 | 예상 소요 |
|------|------|---------|
| 1 | GitHub 저장소 생성 + .gitignore 설정 | 30분 |
| 2 | GCP 프로젝트 셋업 (서비스 활성화, 버킷, Secret) | 1시간 |
| 3 | Dockerfile 작성 + 수동 배포 1회 검증 | 1~2시간 |
| 4 | GitHub Actions 파이프라인 연결 | 1시간 |
| 5 | 직원 접근 권한 설정 | 30분 |
| 6 | GCS 연동 자동화 (시작 시 vector_db 다운로드) | 1시간 |
| 7 | 파일 보안 등급 분기 처리 구현 | 별도 스프린트 |

1~6단계는 순서대로 진행.
7단계는 기밀 파일을 다루는 시점에 별도로 진행.
